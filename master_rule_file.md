# Hey Lola — Master Build Rules

## Role
Act as a senior creative technologist and frontend engineer polishing an already-built premium website for Hey Lola.

## Project state
- The website already exists
- The Restaurant side is largely approved
- The Medical side needs refinement and polish
- Preserve strong existing sections unless there is a clear UX, visual, or conversion reason to improve them
- Do not rebuild from scratch unless explicitly instructed

## Product
Hey Lola is an AI conversational front desk for:
1. Restaurants
2. Medical practices

It handles calls, scheduling flows, FAQs, and escalations, while integrating into real operational systems.

## Core design intent
The site should feel:
- premium
- cinematic
- operationally credible
- elegant, not gimmicky
- more “luxury operating system” than “flashy AI startup”

## Visual language
- Sophisticated, restrained motion
- High-end hospitality aesthetic
- Soft depth, blur, texture, glow used sparingly
- Rounded forms, strong spacing, confident typography
- Avoid generic SaaS boxes and obvious “AI” clichés

## Technical stack
- React 19
- Tailwind CSS
- GSAP / ScrollTrigger where appropriate
- Lucide icons
- Clean, production-grade code only

## Interaction rules
- Every animation must support hierarchy, clarity, or delight
- Avoid motion that feels decorative but pointless
- Hover states should feel premium and tactile
- Preserve performance and readability
- Mobile experience must remain strong

## Medical-side guardrails
- Medical pages must feel calm, clean, and trustworthy
- Never imply diagnosis, triage, or clinical decision-making
- Emphasise scheduling, admin relief, and safe escalation
- The tone should feel more clinical-premium than restaurant-theatrical

## Execution rules
- Polish before replacing
- Improve before reinventing
- Keep the strongest existing structure
- Maintain consistency across Restaurant and Medical
- Build like a world-class product designer, not a template generator
